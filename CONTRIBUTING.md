# 香草前置馆 — 提交规则与免责声明

## 0. 前言与适用范围
本规则（“规则”）适用于通过 GitHub Issue 表单或按本馆流程提交给“香草前置馆”（香草图书馆子页面，以下简称“本馆”或“我们”）的所有条目、索引、说明与关联资料。规则说明收录、审查、编辑、展示、删除与权利义务关系；适用于所有提交者（“提交者”）与网站使用者（“使用者”）。

## 1. 定义
- 提交者：向本馆提交条目内容或信息的个人或组织。
- 资源：被索引或说明的前置、模组、库、数据包、工具、说明文档等实体或链接。
- 条目 / 索引页：基于提交内容在本馆上展示的网页记录。
- 维护团队：负责审核、编辑与发布条目的本馆工作人员或志愿者。

## 2. 提交原则与授权声明
- 提交者应保证对所提交内容拥有合法权利或已获权利人明确授权。
- 提交即表示提交者授予本馆**非独家、全球、免版税、可再授权、可修改（仅为展示/格式化/索引用途）的许可**，用于在本馆及关联服务展示、复制、缓存及索引。
- 提交不转移版权；原权利人可在任何时候提供证明或要求更正/删除（见第9条）。

## 3. 提交者义务（必须遵守）
- **真实性**：提供的信息须真实、准确、可验证。
- **权利与来源**：对非原创内容必须注明来源与原作者，并在可能时提供原始许可证或链接。
- **合法性**：不得提交违法内容（含但不限于侵权、盗版、含恶意软件、侵犯隐私、儿童色情、仇恨言论、煽动暴力等）。
- **隐私保护**：不得在公开提交中包含他人敏感个人信息（身份证号、银行卡号、私人通信内容等）。如确需提交隐私相关证明，请采用私信或邮件方式并标注“仅供审核”。
- **安全声明**：不得谎称已对资源做过安全扫描或扫描结果，若提供扫描报告须附原始证据或可信来源。
- **更新与更正**：若事实变更或发现错误，提交者应及时提出更新请求。

## 4. 必填字段与推荐模板
提交应至少包含所给表单中的必要字段，缺项可能导致延迟或拒绝收录。

## 5. 审核流程与时限（目标时限）
- 提交确认：我们将在收到材料后 **3 个工作日内**在 Issue 中确认已收到或要求补充信息。
- 初步审核：在信息完整情况下，目标在 **7 个工作日内**完成初步合规性检查（版权、明显违法内容、格式）。
- 深度审查与发布：若需更多核实，目标在 **30 日内**完成并决定收录、拒绝或标注为“需验证”。
- 紧急侵权或安全投诉：在收到材料并确认形式完整后，我们将**尽快（通常在 48–72 小时内）**采取临时措施（下架或标注），直至完成调查。

说明：上述时限为目标承诺，视人力和证据复杂度可能延长。我们会在 Issue 中实时更新进度。

## 6. 编辑权与展示规范
- 我们有权为统一格式、可读性与合规性对提交内容进行编辑、合并或删减，但**不会改变事实性声明**（如提交者声明、作者信息、许可条款），除非提交者授权或有明确证据证明错误。
- 我们会在条目中保留来源链接与提交记录；重要改动将在 Issue 中记录变更摘要。
- 对于存在争议或安全风险的条目，我们可能添加显著警告标签（如“未经验证”、“可能包含恶意代码”等）。

## 7. 不担保与免责声明（核心）
- **本馆不对任何被收录资源的功能性、兼容性、可用性或安全性作任何明示或暗示保证。**
- 使用任何被索引或链接的资源，风险由使用者自行承担。
- 在法律允许最大范围内，本馆对因使用、下载或依赖列示内容而导致的任何直接、间接、附带、惩罚性或结果性损害不承担责任。

## 8. 知识产权与侵权投诉流程
申诉要求提交书面材料并在 GitHub Issue 发送。为高效处理，请提供：

针对侵权/下架请求（提交方为权利人或其代理）：
- 侵权作品的描述与权属证明（原始创作证明、版权登记、源码仓库所有权等）。
- 被指侵权条目的 URL / 页面 / 提交说明。
- 具体侵权点说明（逐项列出侵权内容）。
- 请求的处理操作（删除/屏蔽/更正）。
- 联系方式。

反通知（反申诉）要求（若被指侵权方认为非侵权）：
- 身份与联系邮箱、被移除内容的说明、陈述在合理信念下内容不构成侵权的声明、同意接受管辖法院信息并承担可能的法律后果的签名。

处理原则与时限：
- 对于形式完整的侵权投诉，我们将于 **3 个工作日内**确认收到，并在核实必要性后采取临时措施。
- 核查与最终决定依据事实与证据；如需法律意见，我们可能要求权利人通过法院或权威仲裁决定。
- 我们不承担对因错误下架造成的间接损失，但会在合理范围内协助双方沟通与复核。

## 9. 隐私与个人信息处理
- 我们仅为处理条目审核与沟通目的使用提交时提供的联系方式与非敏感个人信息。
- 不在公开页面暴露未经授权的个人敏感信息。
- 若提交包含需受保护的个人信息，请通过私信或邮件并标注“仅供审核”，我们将按适用隐私法规处理。

## 10. 内容删除、修正与争议解决
- 本馆保留删除、屏蔽、合并或归档条目的权利，基于合法性、合规性、重复性或安全性考虑。
- 若对处理结果有异议，可在原 Issue 中提交复审请求，维护团队将在合理时间内回复并说明理由。
- 对无法内部解决的争议，双方可选择仲裁或诉讼，适用法律见第11条。

## 11. 责任限制与法律适用
- 在法律允许最大范围内，本馆对因本服务或被收录内容产生的任何形式的损失或责任予以**限制或排除**。
- 本规则适用并受网站运营者所在地法律管辖；若发生诉讼，通常提交至运营者所在地有管辖权的法院，除非另有约定。

## 12. 其他重要说明
- 非主流/实验性资源可能被标注“未经广泛测试”。请谨慎使用并在沙盒环境中验证。
- 本馆不托管或分发盗版或违反许可的二进制文件；对于提供的第三方下载链接我们不负责其托管方行为。
- 提交不构成对资源的技术支持承诺；如需支持请联系原作者/维护者。

## 13. 联系方式与生效
- 优先渠道：在对应 GitHub 仓库中创建或回复 Issue。
- 本规则自 **生效日期** 起执行，维护团队可随时更新并在页面标注生效/更新日期。提交后若继续使用本馆服务即视为同意最新规则。
