kongbaiyo 制作了一个更为精准的判定系统：家门边的超棒射线追踪系统
与原版的方块选择框判定完全一致，性能占用也并不高，强烈推荐！


目前常用的方块穿透判定，是把可穿透的方块 ID 写到一个 Block Tag 里，然后用 execute unless block ~ ~ ~ #foobar 来判定这个方块是否可穿透。
不过，这种做法其实是不准确的，例如台阶、楼梯、栅栏等方块，它们的模型并不是占据了一整个方块。于是本人编写了一个可复用的 Function 与相关 Tags。

spgoding:is_hit
判定 Function 执行位置是否与方块撞击。
有 #####  注释的内容请在 18w31a 后启用
  1. # spgoding:is_hit
  2. # as A area_effect_cloud.
  3. # return @s spg_return
  4. #         0 - is not hitting any blocks
  5. #         1 - is hitting blocks

  7. # 获取 x 坐标小数点后 1 位
  8. summon minecraft:area_effect_cloud ~ ~ ~ {Duration:2,NoGravity:1b,Tags:["NEW"]}
  9. execute store result score $x spg_mw_tmp run data get entity @e[tag=NEW,limit=1,distance=..1,sort=nearest] Pos[0] 10
  10. scoreboard players operation $x_after_dot spg_mw_tmp = $x spg_mw_tmp
  11. scoreboard players operation $x_after_dot spg_mw_tmp %= 10 const
  12. ##### execute if score $x spg_mw_tmp matches ..-1 run scoreboard players operation $x_after_dot spg_mw_tmp *= -1 const
  13. kill @e[tag=NEW]
  14. # 获取 y 坐标小数点后 1 位
  15. summon minecraft:area_effect_cloud ~ ~ ~ {Duration:2,NoGravity:1b,Tags:["NEW"]}
  16. execute store result score $y spg_mw_tmp run data get entity @e[tag=NEW,limit=1,distance=..1,sort=nearest] Pos[1] 10
  17. scoreboard players operation $y_after_dot spg_mw_tmp = $y spg_mw_tmp
  18. scoreboard players operation $y_after_dot spg_mw_tmp %= 10 const
  19. kill @e[tag=NEW]
  20. # 获取 z 坐标小数点后 1 位
  21. summon minecraft:area_effect_cloud ~ ~ ~ {Duration:2,NoGravity:1b,Tags:["NEW"]}
  22. execute store result score $z spg_mw_tmp run data get entity @e[tag=NEW,limit=1,distance=..1,sort=nearest] Pos[2] 10
  23. scoreboard players operation $z_after_dot spg_mw_tmp = $z spg_mw_tmp
  24. scoreboard players operation $z_after_dot spg_mw_tmp %= 10 const
  25. ##### execute if score $z spg_mw_tmp matches ..-1 run scoreboard players operation $z_after_dot spg_mw_tmp *= -1 const
  26. kill @e[tag=NEW]

  28. execute unless block ~ ~ ~ #spgoding:through run scoreboard players set @s spg_return 1
